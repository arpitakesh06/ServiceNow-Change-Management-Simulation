ServiceNow Change Management Simulation

## 📌 Project Overview

This project demonstrates the complete Change Management lifecycle in ServiceNow based on ITIL 4 practices.

The objective was to resolve a VPN configuration issue while following proper approval, risk assessment, and implementation procedures.

---

## 🔄 Change Lifecycle Covered

* New
* Assess
* Authorize (Approval)
* Scheduled
* Implement
* Review
* Closed

---

## ⚙️ Scenario

A VPN configuration issue was causing authentication failures for users.

A structured change request was created and processed through all ITIL stages to ensure safe resolution without impacting business operations.

---

## ✅ Actions Performed

* Created change request for VPN issue
* Performed risk & impact analysis
* Assigned change to Network team
* Completed approval process
* Scheduled and implemented fix
* Verified resolution in review phase
* Successfully closed the change

---

## 🎯 Outcome

VPN configuration issue resolved successfully with no further impact observed.

Ensured controlled change implementation with minimal risk and zero post-deployment issues.

---

## 🛠 Tools Used

* ServiceNow
* ITIL 4 Framework

---
## 🔗 ITSM Integration

- Incident created for VPN failure  
- Problem identified for root cause analysis  
- Change request raised to implement permanent fix
